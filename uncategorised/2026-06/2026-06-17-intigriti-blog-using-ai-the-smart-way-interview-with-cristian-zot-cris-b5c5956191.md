## Defensive takeaways
- AI should be used as a force multiplier to augment security testing, not replace human judgment
- Always provide AI tools with clear scope definitions to avoid out-of-scope testing and false positives
- Maintain human oversight of AI-generated suggestions and recommendations throughout the security testing process
- Use AI for mechanical tasks (report writing, concept explanation, code analysis) rather than autonomous vulnerability discovery
- Leverage AI for visual explanations and learning complex security concepts like SSRF, RFC specifications, and error code analysis
- Write reports independently first, then use AI only for cleanup and formatting to maintain accuracy and avoid filler content
- Frame AI interactions around specific vulnerability classes to generate more targeted and relevant testing guidance
- Verify all findings independently before sharing with AI to ensure accuracy in report improvement phase
