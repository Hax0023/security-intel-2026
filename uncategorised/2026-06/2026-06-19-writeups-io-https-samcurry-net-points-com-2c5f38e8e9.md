## Summary
Multiple critical vulnerabilities were discovered in Points.com, the backend provider for major airline and hotel rewards programs, affecting 22 million customer records. These included unauthenticated path traversal enabling customer data access, authorization bypasses allowing points transfer with minimal information, and leaked tenant API credentials enabling full administrative access. All vulnerabilities were remediated within hours to days of responsible disclosure.

## Attack scenario (step by step)
1. Attacker discovers unauthenticated HTTP path traversal endpoint on Points.com API
2. Through directory traversal, attacker gains access to internal API querying 22 million order records containing PII and partial credit cards
3. Attacker enumerates customer data using sorting parameters to find specific targets
4. Attacker exploits authorization bypass using only surname and rewards number (both publicly leaked in step 2)
5. Attacker generates full authentication tokens and transfers points to themselves or performs administrative actions
6. Alternatively, attacker discovers and uses leaked tenant credentials (macID/macKey) to sign API requests as legitimate airline on lcp.points.com

## Root cause
Multiple fundamental security design flaws: (1) Path traversal due to improper input validation and lack of access controls on internal APIs, (2) Authorization bypass from weak authentication requiring only surname and rewards number without proper session validation, (3) Hardcoded or improperly managed API credentials stored in client-accessible endpoints, (4) Absence of rate limiting and enumeration protection on sensitive data endpoints

## Attacker mindset
An attacker targeting high-value loyalty programs would recognize Points.com as a central attack surface affecting millions of customers across airlines and hotels. The discovery progression demonstrates methodical reconnaissance: finding path traversal leads to data exposure, which then enables credential stuffing and authorization bypass attacks. Leaked tenant credentials represent the ultimate prize—administrative access to modify points and customer accounts at scale. The attacker likely prioritized disclosure after confirming impact rather than exploitation.

## Defensive takeaways
- Implement strict input validation and canonicalization to prevent path traversal attacks; whitelist allowed API endpoints
- Require multi-factor authentication for authorization, never rely solely on public-facing identifiers like surnames and rewards numbers
- Never embed API credentials in client-side code or discoverable endpoints; use secure credential management systems and rotate credentials regularly
- Implement proper session management with secure tokens that cannot be generated by attackers with limited information
- Apply rate limiting and anomaly detection to API endpoints accessing sensitive customer data
- Conduct regular security audits of internal APIs and ensure they have equivalent security controls to external-facing APIs
- Implement comprehensive logging and alerting for bulk data access or abnormal API usage patterns
- Segment API credentials by least privilege; tenant credentials should have minimal required permissions
- Use API versioning and deprecate legacy endpoints with weak authentication
- Perform regular penetration testing focusing on multi-stage attack chains combining multiple vulnerabilities

## Variant hunting
Researchers should investigate: (1) Other Points.com-hosted loyalty programs for similar credential leakage patterns, (2) Different API endpoints for authorization bypass logic that may accept alternate identity verification combinations, (3) Historical path traversal patterns in Points.com APIs that may have been partially patched, (4) Webhook or callback mechanisms that might leak credentials, (5) Staging/development environments that may expose similar credentials, (6) Third-party integrations with Points.com that inherit these vulnerabilities, (7) Cache poisoning or CORS misconfigurations enabling cross-origin credential exfiltration

## Notes
This writeup demonstrates exceptional responsible disclosure: researchers gave Points.com immediate notice and the team responded within minutes to hours. The cascade of related vulnerabilities suggests a systemic architecture problem rather than isolated bugs. The 22 million affected records and cross-airline scope makes this one of the most impactful loyalty platform breaches. The progression from path traversal → data exposure → credential reuse → administrative access is a textbook example of attack chaining. Notably, the writeup explicitly states full access to global administration console and wallet admin were achieved, though details are redacted, suggesting even more severe impact than disclosed.
